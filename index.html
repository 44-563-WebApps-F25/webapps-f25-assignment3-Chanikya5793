How to Prepare a Simple Git Commit Workflow

  1. Stage your changes
    1. Review modified files in your editor.
    2. Run git status to see tracked vs untracked files.
    3. Stage specific file: git add index.html.
  2. Write a clear commit message
    1. Use present tense (e.g., Add ordered list section).
    2. Keep summary under 50 characters when possible.
  3. Commit the changes
    1. Run git commit -m "Add ordered list section for Part 5".
  4. Push to the remote
    1. Ensure you're on main (check git branch).
    2. Run git push origin main.
    3. Verify on GitHub web interface.
  5. Optional cleanup
    1. Run git log --oneline -5 to confirm history.
    2. Share commit URL if needed.
